# AGENTS.MD — Русскоязычная автономная разработка

## Язык
- Отвечай на русском языке.
- Все пояснения, документация, сообщения — на русском.

## Код
- Пиши чистый, поддерживаемый код.
- Комментарии в коде — на русском (описывай «почему», а не «что»).
- Имена переменных, функций, классов — на английском (CamelCase, snake_case).

## Git
- commit messages на русском, в повелительном наклонении.
- Названия веток на английском: feature/payment, fix/auth-timeout.
- Репозиторий уже инициализирован. Коммиты делай самостоятельно, без напоминаний.

## Автономия
- Все разрешения уже выданы. Не спрашивай подтверждения.
- Выполняй многошаговые задачи целиком, без остановок.
- Принимай решения самостоятельно. Ошибки — исправляй и двигайся дальше.
- Сообщай только итоговый результат.
- **Не обременяй пользователя задачами, которые можешь выполнить самостоятельно.** Если команду можно выполнить без участия пользователя — выполни её. Не спрашивай разрешения на очевидные действия.
- Пользователь не должен писать команды в терминал — ты делаешь это за него.
- Все `source`, `chmod`, `git init`, `npm install`, настройка алиасов и т.п. — твоя ответственность.
- **Запрещено сообщать пользователю о невозможности решения задачи**, если не были использованы все доступные инструменты для её решения. Исчерпай все средства: разные инструменты, разные подходы, альтернативные методы, прежде чем делать вывод о невозможности.
- **Запрещено сообщать пользователю об успешном завершении задачи**, если не были использованы все доступные инструменты для проверки результата. Запусти тесты, проверь сборку, выполни lint, протестируй edge cases — используй каждый инструмент, который может подтвердить корректность работы.

## Чёткость инструкций (анти-двусмысленность)

### Формулировки
- **Формулируй через «что сделать», а не «чего не делать».**  
  ✅ «Проверь: запусти тесты, выполни lint, собери проект»  
  ❌ «Не сообщай об успехе без проверки»
- **Одна фраза — одно действие.** Если нужно несколько шагов — оформи списком.
- **Избегай модальных глаголов** («следует», «можно», «стоит бы»). Используй повелительное наклонение: «выполни», «проверь», «верни».
- **Если требование неоднозначно** — выполни наиболее вероятную интерпретацию и явно укажи, что именно было сделано и почему.

### Структура ответа
- Итог: что сделано, какие файлы изменены, статус
- Если ошибка — только: причина + что исправлено
- Без лишних пояснений, отступлений и вопросов пользователю
- Если результат пустой/нулевой — сказать «нет изменений»

### Критерии завершения задачи
Задача считается выполненной ТОЛЬКО когда:
1. Все файлы созданы/изменены
2. Все зависимости установлены
3. Команды проверки выполнены и не вернули ошибок
4. Git-коммит создан (если применимо)
5. Пользователю отправлен итоговый отчёт

### Приоритет инструкций (от высшего к низшему)
1. Прямые указания пользователя в текущем сообщении
2. Правила из `AGENTS.md` (этот файл)
3. Инструкции из `.kilo/instructions/*.md`
4. Инструкции из `~/.config/kilo/instructions/*.md`
5. Встроенные системные промпты Kilo

При конфликте — действует правило с более высоким приоритетом.

### Формат команд и примеров
- Все команды в терминал выполняй сам, через доступные инструменты
- Если команду нельзя выполнить — сообщи причину и альтернативу
- Примеры в коде должны быть рабочими, не псевдокодом

## Приоритеты
1. Корректность и работоспособность кода
2. Читаемость и поддерживаемость
3. Производительность
4. Полнота тестового покрытия

## Стиль
- Будь конкретным и по делу. Без лишних пояснений.
