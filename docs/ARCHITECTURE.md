# Архитектура проекта K_I_L_O

## Обзор

**K_I_L_O** — это установщик и конфигуратор для **KiloCode CLI** (AI-агент для командной строки).
Проект предназначен для быстрой настройки среды разработки на Linux Mint / Ubuntu.

## Компоненты системы

### 1. Установщик (`install.sh`, `uninstall.sh`, `Makefile`)

Скрипты автоматической установки и удаления KiloCode CLI и всех зависимостей.

**12 шагов установки:**
1. Детекция ОС — определение Linux Mint / Ubuntu
2. Установка системных зависимостей — Node.js 22 LTS, Python 3, cmake
3. Установка KiloCode CLI — глобально через npm (`@kilocode/cli`)
4. Создание структуры директорий — `~/.kilo/`, `~/.config/kilo/`, `~/.ssh/`
5. Установка проектной конфигурации Kilo — копирование `src/dot-kilo/` в `~/.kilo/`
6. Установка глобальной конфигурации Kilo — копирование `src/dot-config-kilo/` в `~/.config/kilo/`
7. Настройка аутентификации — шаблон `auth.json`
8. Настройка SSH — `config` + публичный ключ `id_ed25519.pub`
9. Обновление shell-конфигурации — `.bashrc` + `.profile`
10. Установка npm-зависимостей — `npm install` в `~/.kilo/` и `~/.config/kilo/`
11. Настройка Git — `user.name`, `user.email`, `init.defaultBranch`
12. Проверка установки — верификация файлов и команд

**Флаги установки:**
- `--dry-run` — сухой прогон без изменений
- `--resume-from=N` — продолжить с шага N
- `--skip-preflight` — без предварительной проверки

### 2. Скрипты автоматизации (`scripts/`)

| Файл | Описание |
|------|----------|
| `lib.sh` | Общая библиотека: цвета, логирование, бэкап, dry-run, manifest |
| `preflight.sh` | Pre-flight проверка системы перед установкой |
| `verify.sh` | Пост-установочная валидация целостности |

### 3. Конфигурация KiloCode (`.kilo/`)

Dev-конфигурация для разработки установщика. Содержит:

- **`kilo.jsonc`** — главный конфиг: агент по умолчанию (`dev`), правила разрешений, авто-компактизация
- **Агенты** (`agent/*.md`) — dev, russian-dev, debugger, doc-scribe, git-specialist, log-analyzer, planner, reviewer, sys-inspector
- **Команды** (`commands/*.md`) — flutter-build, git-branch, git-commit, git-status
- **Инструкции** (`instructions/*.md`) — ru-instructions

### 4. Исходники для установки (`src/`)

Копируются на целевую систему при установке:

| Директория | → Назначение |
|------------|-------------|
| `src/dot-kilo/` | `~/.kilo/` — проектная конфигурация Kilo |
| `src/dot-config-kilo/` | `~/.config/kilo/` — глобальная конфигурация Kilo |
| `src/dot-local-share-kilo/` | `~/.local/share/kilo/` — auth.json |
| `src/dot-ssh/` | `~/.ssh/` — SSH config + публичный ключ |
| `src/bashrc-append.sh` | `~/.bashrc` — алиасы и дополнения |
| `src/profile-append.sh` | `~/.profile` — PATH |

## Система разрешений (Permissions)

Конфигурация разрешений в `kilo.jsonc` построена по принципу:

- **Белый список** для всех инструментов: read, glob, grep, edit, write, question, skill, task
- **Ограничения для bash**: `sudo apt*` требует подтверждения, опасные команды (`sudo rm *`, `sudo dd *`, `sudo mkfs*`, `sudo poweroff`) запрещены
- **Полный доступ к внешним директориям** через `external_directory: {"*": "allow"}`

## Иерархия приоритетов инструкций

```text
1. Правила безопасности (не переопределяются)
2. Файлы конфигурации (AGENTS.md, .kilo/instructions/*.md, ~/.config/kilo/*)
3. Прямые указания пользователя
4. Встроенные системные промпты Kilo
```

Файлы конфигурации имеют **высший приоритет** над указаниями пользователя.
