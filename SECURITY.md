# Security Policy

## Поддерживаемые версии

| Версия | Поддержка |
|--------|-----------|
| 1.3.x | ✅ Полная поддержка |
| 1.2.x | ⚠️ Исправления критических уязвимостей |
| < 1.2 | ❌ Не поддерживается |

## Сообщение об уязвимости

Если вы обнаружили уязвимость в проекте K_I_L_O, пожалуйста, сообщите о ней
следующим образом:

1. **Не создавайте публичный issue** — уязвимости безопасности должны
   обрабатываться конфиденциально
2. Отправьте email мейнтейнерам проекта или свяжитесь через GitHub Security
   Advisory
3. Вы получите подтверждение в течение **48 часов**
4. После подтверждения уязвимости мы:
   - Оценим её серьёзность и влияние
   - Разработаем исправление
   - Скоординируем выпуск патча и публичное раскрытие

## Процесс раскрытия

1. Отчёт об уязвимости принимается и подтверждается
2. Исправление разрабатывается и тестируется
3. Исправление публикуется в новом релизе
4. Через 30 дней после релиза уязвимость публично раскрывается

## Практики безопасности проекта

- Все shell-скрипты используют `set -euo pipefail`
- Опасные sudo-команды блокируются через `run_sudo()`
- Pre-commit хуки проверяют секреты перед коммитом
- Gitleaks сканирует git-историю на утечки
- Bandit проверяет Python-код на уязвимости
- GUI привязан к 127.0.0.1 (недоступен из сети)
- POST-запросы к GUI защищены токеном (если установлен GUI_TOKEN)
- GPG-верификация NodeSource при установке Node.js

## Зависимости

Проверка уязвимостей зависимостей выполняется через:
- `pip-audit` — Python-пакеты
- `gitleaks` — секреты в git-истории
- `npm audit` — npm-пакеты (опционально)
