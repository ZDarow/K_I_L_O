# Агенты KiloCode

Проект включает **9 агентов** для KiloCode CLI. Ниже — полный каталог с описанием каждого.

---

## Primary-агенты

### 1. `dev` — Универсальный агент разработки (по умолчанию)

| Свойство | Значение |
|----------|----------|
| **Тип** | primary |
| **Шагов** | 200 |
| **Цвет** | #0066CC |

Универсальный агент с полной автономией. Специализация: Linux, CI/CD, автоматизация, общее ПО.

**Полномочия:** все разрешения выданы, не требует подтверждения.

**Протокол автонастройки:**
1. Детекция платформы (uname, python3, git)
2. Установка базовых инструментов
3. Настройка платформы по задаче

### 2. `git-specialist` — Git-специалист

| Свойство | Значение |
|----------|----------|
| **Тип** | primary |
| **Шагов** | 150 |
| **Цвет** | #E65100 |

Управление репозиториями, ветвление, коммиты, слияния, CI/CD.

**Правила:**
- Коммиты на русском, в повелительном наклонении
- Ветки на английском, kebab-case
- `git push`, `git commit`, `git merge --force` — требуют подтверждения

## Subagent-агенты

### 3. `debugger` — Debugger

Анализ ошибок, стектрейсов, поиск корневых причин:
- Парсинг error-сообщений и стектрейсов
- Анализ логов, корреляция событий
- 5 Whys техника, cause-effect mapping
- Воспроизведение и изоляция багов

### 4. `doc-scribe` — Technical Writer

Создание и поддержка документации:
- README, CONTRIBUTING, API-спецификации
- Архитектурные ADR
- Mermaid-диаграммы
- Форматы: Markdown, AsciiDoc, reStructuredText

### 5. `log-analyzer` — Log Analyzer

Анализ логов: парсинг, фильтрация, агрегация:
- Поиск паттернов ошибок, временной анализ
- Поддержка форматов: plain text, JSON-lines, syslog
- Стектрейсы: извлечение и группировка
- Инструменты: grep, rg, journalctl, zcat

### 6. `planner` — Planner Specialist

Планирование реализации задач с выбором подхода:
- **STANDARD** — базовое планирование
- **TWO** — сравнение двух подходов
- **HARD** — сложные исправления (баги, race conditions, утечки памяти)
- **CRO** — конвертация/рефакторинг
- **CI** — CI/CD настройка

### 7. `reviewer` — Code Reviewer

Ревью кода: качество, безопасность, производительность, архитектура.

### 8. `sys-inspector` — System Inspector

Инспекция системы Linux: процессы, диски, сеть, память, логи, производительность.

### 9. `russian-dev` — Русскоязычный ассистент разработки (DEPRECATED)

Перенесён в `dev.md`. Используйте `/agent dev`.

## Типы агентов

| Тип | Описание |
|-----|----------|
| **primary** | Полноценный агент, отображается в списке агентов,有自己的 шаги и цвет |
| **subagent** | Вспомогательный агент, вызывается другими агентами, не отображается в основном списке |
| **deprecated** | Устаревший агент, функциональность перенесена в `dev` |

---

## Использование агентов

```bash
# Вызов агента по умолчанию (dev)
npx kilo

# Вызов конкретного агента
npx kilo --agent debugger
# или
/agent debugger

# Вызов subagent
/agent debugger "Проанализируй ошибку в логах"
```
