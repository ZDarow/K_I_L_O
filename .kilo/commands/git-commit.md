# Git Commit

1. Показать изменения:
   ```bash
   git diff --stat
   git diff
   ```
2. Добавить файлы: `git add -A` (или выборочно)
3. Создать коммит:
   ```bash
   git commit -m "<тип>: <описание>"
   ```
4. **Подтверждение пользователя обязательно** перед коммитом и пушем

## Типы коммитов
- `feat:` — новая функциональность
- `fix:` — исправление ошибки
- `refactor:` — рефакторинг
- `docs:` — документация
- `test:` — тесты
- `chore:` — обслуживание (зависимости, конфиги)
- `style:` — форматирование кода
- `perf:` — оптимизация производительности
- `ci:` — CI/CD

## Примеры
```
feat: добавлять сохранение настроек в SharedPreferences
fix: исправлять расчёт контрольной суммы для команд громкости
docs: описывать BLE-протокол Prology
```
